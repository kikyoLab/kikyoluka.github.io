    • Css position fixed 打印重复内容

    pixiv
    热度0℃记事本笔记

    前言

    最近在敲一个在线条码设计的demo时发现了一个问题，因为demo里的所有元素都可以拖拽，所以对所有元素的css
    进行了设定，如position：fixed等等. 在后续调试中发现，调用window.print()对DIV框内所有元素进行打印中会
    产生内容重复打印的问题，比如第二页多出了第一页的内容，第三页多出了第一页第二页的内容. 上MDN查了后才发现，
    MDN 对 position:fixed的描述：

    fixed

    元素会被移出正常文档流，并不为元素预留空间，而是通过指定元素相对于屏幕视口（viewport）的位置来指定元素位置，元素的位置在屏幕滚动时不会改变. 打印时，元素会出现在的每页的固定位置（也就是说每一页的固定位置都会出现，所以内容会重复打印）.fixed 属性会创建新的层叠上下文，当元素祖先的 transform, perspective 或 filter 属性非 none 时，容器由视口改为该祖先.

    实践

    问题已经找到了，那么就该去解决了.
    如果你没有拖拽需求的话，可以直接将position:fixed进行修改，比如absolute等等，但是有拖拽需求的话就会出问题，你会发现元素在拖拽过程中错位了.

    解决

    所以我决定在触发打印按钮时将DIV框内所有打印元素的position:fixed修改为position:absolute，然后你会发现整体元素向右下进行了位移，但是在打印预览框内显示的是正常的布局，打印出来的内容也是正常的. 而且，可以在调用打印事件后再将打印元素的position修改回fixed，这样在整个打印流程中打印元素不会进行位移，预览界面也不会出现内容重复的问题，像我下面的代码一样：

    /* print */
Print.onclick = function () {
    $("#printmain").children().css('position', 'absolute')

    printJS({
        printable: "printmain",
        type: "html",
        css: '/css/print.css',
        scanStyles: false
    });

    $("#printmain").children().css('position', 'fixed')
}
